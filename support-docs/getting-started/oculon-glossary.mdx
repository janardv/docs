

## Variables
A variable is a named value that represents a metric, assumption, or calculation used to build and drive your financial model.

## Helper Variables
Helper variables are intermediate variables used to simplify complex formulas by breaking them into smaller, reusable steps.

## Actual Formuala
An actual formula defines how values are calculated using real, observed data within the model.

## Plan Formuala
A plan formula defines how future or projected values are calculated based on formula, assumptions, drivers, or scenarios.

## Time Cell values
A time value represents the value of a variable for a specific time period (e.g., a month or quarter) within the model.

## Models
A model is a structured workspace where variables, formulas, and data are organized to plan, analyze, and forecast business outcomes.

## Sections
A section is a logical grouping of related variables within a model, used to organize and structure your analysis.

## Trend
Trend shows a visual representation of how a value changes over a selected time range within the model.

## Charts

#### Bar Chart
A bar chart displays data using bars to compare values across categories or periods.

#### Line Chart
A line chart displays data as a continuous line to show trends and changes over time.

#### KPI Card
A KPI card displays a key metric with its value, helping you quickly track performance at a glance.

## Dimensions
Dimensions are categories used to segment and organize data, such as by product, region, or customer.

## Boards
Boards are visual interfaces used to explore, present, and monitor data through charts, tables, and key metrics.

## Database
The database is where raw and structured data is stored, managed, and queried for use in models and boards.

## Integrations
Integrations are connections to external data sources that allow you to import, sync, and query data into your models and database.

## Granularity
Granularity determines the time interval at which data is represented in the model, such as monthly, quarterly, or yearly.

## Scenarios

## Library
The library is a centralized place to manage shared resources such as integrations and dimensions used across models.

## Templates
Templates are reusable setups used to quickly create models, boards, databases, or combinations of these with predefined structure and logic.

## Spaces
Spaces are collaborative environments used to organize and manage models, boards, and databases for teams or specific use cases.

## Private space
A private space is a default personal space that cannot be shared with others and is accessible only to you.

## Workspace
A workspace is the top-level environment that contains all your spaces, models, boards, and data, and represents a distinct organization or account.

## Model Date Range
The model date range defines the start and end periods over which the model is built and analyzed.

## Last Actual Date
The last actual date marks the most recent period for which real (actual) data is available, separating actuals from planned values.

## Relative Date
Relative date allows time periods to adjust dynamically based on the current date instead of fixed calendar dates.(Month 1, Month 2,..)

## Fiscal year
Fiscal year defines the accounting year used in the model, which may differ from the calendar year.

## Queries
A query is used to fetch and transform data from connected integrations to create and populate databases within Oculon.

## Linked Models
Linked models are models that are referenced within the current model through formulas or dependencies.
